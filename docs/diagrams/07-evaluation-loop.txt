================================================================================
              EVALUATION LOOP — Round Enforcement Logic
================================================================================

  Every phase uses the same evaluation loop pattern with configurable
  eval.min_rounds and eval.max_rounds (phase-scoped in .forgectl/config)
  to balance quality with progress.


  ═══════════════════════════════════════════════════════════════════════════
  GENERIC EVALUATION LOOP
  ═══════════════════════════════════════════════════════════════════════════

                          ┌──────────────┐
                          │  WORK DONE   │
                          │  (Draft /    │
                          │   Implement) │
                          └──────┬───────┘
                                 │
                                 ▼
                          ┌──────────────┐
                     ┌───►│  EVALUATE    │◄──────────────────────┐
                     │    │              │                        │
                     │    │  rounds++    │                        │
                     │    └──────┬───────┘                        │
                     │           │                                │
                     │    ┌──────┴──────────────────────┐        │
                     │    │                             │        │
                     │    ▼                             ▼        │
                     │  PASS                          FAIL       │
                     │    │                             │        │
                     │    │                             │        │
                     │    ▼                             ▼        │
                     │  rounds >= eval.min_rounds? rounds >= eval.max_rounds?
                     │    │                             │        │
                     │  ┌─┴──┐                       ┌──┴─┐     │
                     │  │    │                       │    │     │
                     │ YES   NO                    YES   NO     │
                     │  │    │                       │    │     │
                     │  │    ▼                       │    ▼     │
                     │  │  REFINE ───────────────────┼────┘     │
                     │  │  (must meet minimum)       │          │
                     │  │                            │          │
                     │  ▼                            ▼          │
                     │  ACCEPT                    ACCEPT        │
                     │  (quality met)          (forced — prevent│
                     │                          infinite loops) │
                     │                                          │
                     └──────── REFINE ──────────────────────────┘
                          (fix deficiencies)


  ═══════════════════════════════════════════════════════════════════════════
  DECISION MATRIX
  ═══════════════════════════════════════════════════════════════════════════

  ┌───────────┬────────────────────────┬────────────────────────────────────┐
  │ Verdict   │ Round Condition         │ Outcome                           │
  ├───────────┼────────────────────────┼────────────────────────────────────┤
  │ PASS      │ rounds >= eval.min     │ ACCEPT  (quality satisfied)       │
  │ PASS      │ rounds <  eval.min     │ REFINE  (need more eval cycles)   │
  │ FAIL      │ rounds <  eval.max     │ REFINE  (fix and retry)           │
  │ FAIL      │ rounds >= eval.max     │ ACCEPT  (forced — cap reached)    │
  └───────────┴────────────────────────┴────────────────────────────────────┘


  ═══════════════════════════════════════════════════════════════════════════
  EXAMPLES BY CONFIGURATION
  ═══════════════════════════════════════════════════════════════════════════

  Example 1: [phase].eval.min_rounds = 1, max_rounds = 3 (typical)
  ───────────────────────────────────────────────────────────────

    Round 1: EVALUATE ──► PASS ──► ACCEPT (done in 1 round)
    Round 1: EVALUATE ──► FAIL ──► REFINE
    Round 2: EVALUATE ──► PASS ──► ACCEPT (fixed in round 2)
    Round 2: EVALUATE ──► FAIL ──► REFINE
    Round 3: EVALUATE ──► FAIL ──► ACCEPT (forced at max)


  Example 2: [phase].eval.min_rounds = 2, max_rounds = 3 (high quality)
  ────────────────────────────────────────────────────────────────────

    Round 1: EVALUATE ──► PASS ──► REFINE (below min, must continue)
    Round 2: EVALUATE ──► PASS ──► ACCEPT (met minimum)
    Round 1: EVALUATE ──► FAIL ──► REFINE
    Round 2: EVALUATE ──► FAIL ──► REFINE
    Round 3: EVALUATE ──► FAIL ──► ACCEPT (forced at max)


  Example 3: [phase].eval.min_rounds = 1, max_rounds = 1 (fast pass)
  ───────────────────────────────────────────────────────────────────

    Round 1: EVALUATE ──► PASS ──► ACCEPT
    Round 1: EVALUATE ──► FAIL ──► ACCEPT (forced, no retry)

  ═══════════════════════════════════════════════════════════════════════════
  enable_eval_output CONFIGURATION
  ═══════════════════════════════════════════════════════════════════════════

  The per-phase enable_eval_output config flag controls whether eval sub-agents
  write reports to files or communicate verdicts verbally.
  Configured at: [specifying.eval], [planning.eval], [implementing.eval]

  When enable_eval_output: true (not default)
  ────────────────────────────────────────
  - EVALUATE output includes: "After completion, advance with
    --verdict PASS|FAIL --eval-report <path>"
  - REFINE output includes the eval report path to study
  - forgectl advance requires --eval-report <path>
  - Report written to disk and stored as reference

  When enable_eval_output: false
  ──────────────────────────────
  - EVALUATE output includes: "After completion, advance with --verdict PASS|FAIL"
  - REFINE output says "Make corrections based off communication with the evaluator"
  - forgectl advance requires only --verdict (no --eval-report)
  - No report file written; verdict communicated directly by sub-agent

  Both modes:
  - forgectl eval outputs full evaluator context (embedded evaluator prompts)
  - --eval-report flag accepted but ignored when enable_eval_output: false
    (warning printed: "--eval-report is ignored, eval output is not enabled")
