               PHASE 3: PLANNING — Detailed State Machine
================================================================================

  Role: Staff Engineer
  Input: plan-queue.json + specs/*.md
  Output: plan.json + notes/*.md (implementation guidance per package)
  Evaluator: 11-dimension plan review by AI sub-agent

                                   ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │  EVALUATE                                                               │
  │                                                                         │
  │  Command: forgectl eval  (outputs full context for sub-agent)           │
  │                                                                         │
  │  Sub-agent assesses plan against 11 dimensions:                         │
  │                                                                         │
  │    1. Behavior         — plan items cover all spec behaviors            │
  │    2. Error Handling   — failure modes mapped to tests                  │
  │    3. Rejection        — invalid inputs tested                          │
  │    4. Interface        — outputs match spec contracts                   │
  │    5. Configuration    — configurable params covered                    │
  │    6. Observability    — logging requirements addressed                 │
  │    7. Integration      — cross-spec dependencies mapped                │
  │    8. Invariants       — always-true properties enforced               │
  │    9. Edge Cases       — boundary conditions tested                    │
  │   10. Testing Criteria — all tests mapped to plan items                │
  │   11. Dependencies     — DAG valid, schema compliant                   │
  │                                                                         │
  │  Output: evals/round-N.md  (when enable_eval_output: true)              │
  │                                                                         │
  │  Action (enable_eval_output: true):                                     │
  │    forgectl advance --verdict PASS|FAIL --eval-report <path>            │
  │  Action (enable_eval_output: false):                                    │
  │    forgectl advance --verdict PASS|FAIL                                 │
  └────────────────┬──────────────────────────────┬─────────────────────────┘
                   │                              │
