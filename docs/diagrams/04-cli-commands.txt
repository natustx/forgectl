================================================================================
                    FORGECTL CLI — Command Reference Map
================================================================================

  Binary: forgectl (Go, Cobra framework)
  State:  .forgectl/state/forgectl-state.json (JSON-backed state machine)


  forgectl
  │
  ├── init ─────────────── Initialize a new session
  │   │
  │   │  Required flags:
  │   │    --from <path>  Input file (spec-queue, plan-queue, or plan.json)
  │   │    --phase <str>  Phase: specifying | planning | implementing
  │   │
  │   │  Optional flags:
  │   │    --guided       Enable guided mode (user checkpoints)
  │   │    --no-guided    Disable guided mode
  │   │
  │   │  Configuration:
  │   │    All config (batch, eval.min_rounds, eval.max_rounds) read from
  │   │    .forgectl/config (TOML). Phase-scoped:
  │   │      [specifying]
  │   │        batch = 1
  │   │        [specifying.eval]
  │   │          min_rounds = 1
  │   │          max_rounds = 3
  │   │      [general]
  │   │        enable_commits = false
  │   │
  │   │  Input by phase:
  │   │    specifying   ──►  spec-queue.json
  │   │    planning     ──►  plan-queue.json
  │   │    implementing ──►  plan.json
  │   │
  │   │  State file location: .forgectl/state/forgectl-state.json
  │   │
  │   └─── Output: state initialized
  │
  │
  ├── advance ──────────── Transition to next state
  │   │
  │   │  Context-dependent flags:
  │   │
  │   │  ┌─────────────────┬─────────────────────────────────────────────┐
  │   │  │ Current State    │ Valid Flags                                 │
  │   │  ├─────────────────┼─────────────────────────────────────────────┤
  │   │  │ ORIENT           │ (none required)                            │
  │   │  │ SELECT           │ (none required)                            │
  │   │  │ DRAFT            │ --file <path>  (override output, spec only)│
  │   │  │ EVALUATE         │ --verdict PASS|FAIL                        │
  │   │  │                  │ --eval-report <path>  (if enable_eval_out) │
  │   │  │ REFINE           │ (none required)                            │
  │   │  │ ACCEPT           │ --message (planning, if enable_commits)    │
  │   │  │ IMPLEMENT        │ --message (first round, if enable_commits) │
  │   │  │ COMMIT           │ --message (if enable_commits)              │
  │   │  │ RECONCILE        │ (none required)                            │
  │   │  │ RECONCILE_EVAL   │ --verdict PASS|FAIL  --eval-report <path>  │
  │   │  │ RECONCILE_REVIEW │ (none required)                            │
  │   │  │ COMPLETE         │ --message (specifying, if enable_commits)  │
  │   │  │ PHASE_SHIFT      │ --from <path>  (specifying or gen_plan_q)  │
  │   │  │ SELF_REVIEW      │ (none required)                            │
  │   │  │ STUDY_*          │ (none required)                            │
  │   │  │ REVIEW           │ (none required)                            │
  │   │  │ VALIDATE         │ (none required)                            │
  │   │  └─────────────────┴─────────────────────────────────────────────┘
  │   │
  │   │  Mode toggles (any state):
  │   │    --guided / --no-guided
  │   │
  │   └─── Output: Updated state + formatted status
  │
  │
  ├── status ──────────── Print current state with action guidance
  │   │
  │   │  No additional flags (inherits --dir from root)
  │   │
  │   └─── Output: Human-readable status, next action, overview
  │
  │
  ├── eval ────────────── Output evaluation context for sub-agent
  │   │
  │   │  Valid only in EVALUATE states
  │   │  No additional flags
  │   │
  │   └─── Output: Full context needed by AI evaluator
  │                (items, specs, tests, evaluator prompt)
  │
  │
  ├── add-queue-item ───── Append a spec to the specifying queue
  │   │
  │   │  Required flags:
  │   │    --name <str>    Spec name (must be unique)
  │   │    --topic <str>   Topic of concern
  │   │    --file <path>   Path to spec file (must exist, must match
  │   │                    <domain>/specs/<name>.md convention)
  │   │
  │   │  Optional flags:
  │   │    --domain <str>  Domain name (required when no match found)
  │   │
  │   │  Domain resolved from: configured domains → session domains
  │   │  Valid in: DRAFT, CROSS_REFERENCE_REVIEW, DONE, RECONCILE_REVIEW
  │   │
  │   └─── Output: "Added {name} to specifying queue for domain {domain}."
  │
  │
  ├── set-roots ────────── Set code search roots for a domain
  │   │
  │   │  Required flags:
  │   │    --domain <str>  Domain name
  │   │    --roots <str>   Comma-separated list of root paths
  │   │
  │   │  Writes to specifying.domains[<domain>].code_search_roots
  │   │
  │   └─── Output: "Set code search roots for {domain}."
  │
  │
  └── --version ───────── Display version (v0.0.1)

  ┌─────────────────────────┐         ┌─────────────────────────┐
  │    ENGINEER              │         │    SUB-AGENT (AI)        │

  │                          │         │                          │
  │  Uses:                   │         │  Uses:                   │
  │    forgectl init         │         │    forgectl eval         │
  │    forgectl advance      │         │                          │
  │    forgectl status       │         │  Reads:                  │
  │    forgectl add-queue-   │         │    - Spec files          │
  │             item         │         │    - Implementation      │
  │    forgectl set-roots    │         │    - Notes/refs          │
  │                          │         │    - Evaluator prompt    │
  │  Writes:                 │         │                          │
  │    - Specs               │         │  Writes:                 │
  │    - Plans               │         │    - Eval reports        │
  │    - Code                │         │    (evals/*.md)          │
  │    - Notes               │         │                          │
  └──────────┬───────────────┘         └──────────┬───────────────┘
             │                                     │
             │    forgectl advance                  │   forgectl eval
             │    --verdict --eval-report           │
             │                                     │
             └──────────────┬──────────────────────┘
                            │
                            ▼
                     ┌──────────────────────────────┐
                     │  .forgectl/state/            │

                     └──────────────────────────────┘
