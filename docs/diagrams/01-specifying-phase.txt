================================================================================
              PHASE 1: SPECIFYING — Detailed State Machine
================================================================================

  Role: Systems Architect
  Input: spec-queue.json (list of specs to draft)
  Output: specs/*.md (permanent behavioral contracts)
  Evaluator: 11-dimension adversarial review by AI sub-agent

  ┌─────────────────────────────────────────────────────────────────────────┐
  │  INITIALIZATION                                                         │
  │                                                                         │
  │  forgectl init \                                                        │
  │    --phase specifying \                                                 │
  │    --from spec-queue.json                                               │
  │                                                                         │
  │  (Config: batch, eval.max_rounds, eval.min_rounds in .forgectl/config)  │
  │  (Optional: --guided for user checkpoints)                              │
  └────────────────────────────────┬────────────────────────────────────────┘
                                   │
                                   ▼
  ╔═══════════════════════════════════════════════════════════════╗
  ║  PER-SPEC LOOP  (repeats for each spec in queue)             ║
  ╠═══════════════════════════════════════════════════════════════╣
  ║                                                               ║
  ║  ┌─────────────────────────────────────────────────┐         ║
  ║  │  ORIENT                                          │         ║
  ║  │                                                  │         ║
  ║  │  - Read planning sources referenced by spec      │         ║
  ║  │  - Review existing specs in project              │         ║
  ║  │  - Understand domain landscape                   │         ║
  ║  │  - If guided: discuss scope with user             │         ║
  ║  │                                                  │         ║
  ║  │  Action: forgectl advance                        │         ║
  ║  └──────────────────────┬──────────────────────────┘         ║
  ║                         ▼                                     ║
  ║  ┌─────────────────────────────────────────────────┐         ║
  ║  │  SELECT                                          │         ║
  ║  │                                                  │         ║
  ║  │  - Pull next spec from queue                     │         ║
  ║  │  - Display: name, domain, topic, file path       │         ║
  ║  │  - Display: planning sources & dependencies      │         ║
  ║  │                                                  │         ║
  ║  │  Action: forgectl advance                        │         ║
  ║  └──────────────────────┬──────────────────────────┘         ║
  ║                         ▼                                     ║
  ║  ┌─────────────────────────────────────────────────┐         ║
  ║  │  DRAFT                                           │         ║
  ║  │                                                  │         ║
  ║  │  Write spec file following spec-format.md:       │         ║
  ║  │                                                  │         ║
  ║  │  ┌─────────────────────────────────────┐        │         ║
  ║  │  │ # Title                              │        │         ║
  ║  │  │ ## Topic of Concern                  │        │         ║
  ║  │  │ ## Context                           │        │         ║
  ║  │  │ ## Depends On                        │        │         ║
  ║  │  │ ## Integration Points                │        │         ║
  ║  │  │ ## Interface                         │        │         ║
  ║  │  │   ### Inputs / Outputs / Rejection   │        │         ║
  ║  │  │ ## Behavior                          │        │         ║
  ║  │  │   ### Preconditions / Steps /        │        │         ║
  ║  │  │       Postconditions / Error Handling│        │         ║
  ║  │  │ ## Configuration                     │        │         ║
  ║  │  │ ## Observability                     │        │         ║
  ║  │  │ ## Invariants                        │        │         ║
  ║  │  │ ## Edge Cases                        │        │         ║
  ║  │  │ ## Testing Criteria                  │        │         ║
  ║  │  │ ## Implements                        │        │         ║
  ║  │  └─────────────────────────────────────┘        │         ║
  ║  │                                                  │         ║
  ║  │  Action: forgectl advance [--file <path>]        │         ║
  ║  └──────────────────────┬──────────────────────────┘         ║
  ║                         ▼                                     ║
  ║  ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐        ║
  ║  │  EVALUATION LOOP (specifying.eval.min..max)      │        ║
  ║  │                                                   │        ║
  ║  │  ┌─────────────────────────────────────────┐     │        ║
  ║  │  │  EVALUATE                                │     │        ║
  ║  │  │                                          │     │        ║
  ║  │  │  Spawn Opus sub-agent to review spec batch │     │        ║
  ║  │  │  against 11 dimensions:                    │     │        ║
  ║  │  │                                            │     │        ║
  ║  │  │   1. Behavior coverage                     │     │        ║
  ║  │  │   2. Error handling                        │     │        ║
  ║  │  │   3. Rejection cases                       │     │        ║
  ║  │  │   4. Interface (inputs/outputs)            │     │        ║
  ║  │  │   5. Configuration                         │     │        ║
  ║  │  │   6. Observability/logging                 │     │        ║
  ║  │  │   7. Integration points                    │     │        ║
  ║  │  │   8. Invariants                            │     │        ║
  ║  │  │   9. Edge cases                            │     │        ║
  ║  │  │  10. Testing criteria                      │     │        ║
  ║  │  │  11. Dependencies & format                 │     │        ║
  ║  │  │                                            │     │        ║
  ║  │  │  Output: <domain>/specs/.eval/batch-N-rN.md│     │        ║
  ║  │  │                                          │     │        ║
  ║  │  │  Action: forgectl advance \              │     │        ║
  ║  │  │    --verdict PASS|FAIL \                 │     │        ║
  ║  │  │    --eval-report <path>                  │     │        ║
  ║  │  └─────────────┬──────────┬────────────────┘     │        ║
  ║  │                │          │                       │        ║
  ║  │          FAIL  │          │  PASS                 │        ║
  ║  │        <max    │          │  >=min                │        ║
  ║  │                ▼          │                       │        ║
  ║  │  ┌──────────────────┐    │                       │        ║
  ║  │  │  REFINE           │    │                       │        ║
  ║  │  │                   │    │                       │        ║
  ║  │  │  Fix deficiencies │    │                       │        ║
  ║  │  │  from eval report │    │                       │        ║
  ║  │  │                   │    │                       │        ║
  ║  │  │  Action: advance  │    │                       │        ║
  ║  │  └────────┬─────────┘    │                       │        ║
  ║  │           │               │                       │        ║
  ║  │           └───► EVALUATE  │                       │        ║
  ║  │                           │                       │        ║
  ║  └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┼ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘        ║
  ║                              ▼                                ║
  ║  ┌─────────────────────────────────────────────────┐         ║
  ║  │  ACCEPT                                          │         ║
  ║  │                                                  │         ║
  ║  │  Spec finalized. No commit here —                │         ║
  ║  │  single commit occurs at COMPLETE.               │         ║
  ║  │                                                  │         ║
  ║  │  Action: forgectl advance                        │         ║
  ║  └──────────────────────┬──────────────────────────┘         ║
  ║                         │                                     ║
  ║            ┌────────────┴────────────┐                        ║
  ║            │                         │                        ║
  ║       more specs              queue empty                     ║
  ║            │                         │                        ║
  ║            ▼                         ▼                        ║
  ║       ► ORIENT                    DONE                        ║
  ║         (next spec)                  │                        ║

                                         │
                                         ▼
  ╔═══════════════════════════════════════════════════════════════╗

  ╠═══════════════════════════════════════════════════════════════╣
  ║                                                               ║
  ║  ┌─────────────────────────────────────────────────┐         ║
  ║  │  RECONCILE                                       │         ║

  ║  │  - Naming consistency across all specs           │         ║
  ║  │                                                  │         ║
  ║  │  Action: forgectl advance                        │         ║
  ║  └──────────────────────┬──────────────────────────┘         ║
  ║                         ▼                                     ║
  ║  ┌─────────────────────────────────────────────────┐         ║
  ║  │  RECONCILE_EVAL                                  │         ║

  ║  │  specifying.reconciliation.max_rounds (def: 3)   │         ║
