================================================================================
                     FORGECTL — FULL DEVELOPMENT LIFECYCLE
================================================================================

  A spec-driven development harness that compiles planning documents into
  production code through four sequential phases, each governed by a state
  machine with iterative AI-powered evaluation loops.

  ═══════════════════════════════════════════════════════════════════════════
  KEY
  ═══════════════════════════════════════════════════════════════════════════

  ──►  State transition (forgectl advance)
  │    Sequential flow
  ◄──  Loop back / retry

  PHASE_SHIFT    Hard stop between phases and at domain boundaries;
                 engineer refreshes context before proceeding
  EVALUATE       AI sub-agent reviews work against evaluation dimensions
  REFINE         Fix deficiencies found in evaluation
  COMMIT         Hard stop; must commit (or advance) before continuing
  GUIDED         Optional pause for human discussion (--guided flag)

  Round Enforcement (from .forgectl/config [phase].eval):
    eval.min_rounds   PASS below minimum forces another REFINE cycle
    eval.max_rounds   FAIL at maximum forces acceptance (prevents infinite loops)

  Mode Control (general config):
    plan_all_before_implementing: false (default)
      Interleaved: plan domain → implement domain → plan domain → ...
    plan_all_before_implementing: true
      All planning first, then all implementing (PHASE_SHIFT at each domain boundary)

  Config & State:
    .forgectl/config                  TOML config file (phase-scoped)
    .forgectl/state/forgectl-state.json State file (JSON)
